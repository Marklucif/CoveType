本機處理

解決真實問題

擺脫雲端 AI 語音輸入的私隱與工作流程負擔。

雲端語音輸入可能暴露敏感內容、增加網絡延遲、輸出凌亂文字，並與常用快捷鍵衝突。CoveType 針對這些問題設計。

01

擔心語音私隱？

語音辨識與 AI 潤飾均在 Mac 本機執行，日常口述不會傳送至雲端轉錄服務。

02

不想等待雲端結果？

本機模型省去遠端 ASR 與寫作 API 往返，離線時仍可使用。

03

原始轉錄太凌亂？

可選本機 Qwen 模型在輸入前改善標點、斷句與口語表達。

04

快捷鍵經常誤觸？

可記錄單鍵或組合鍵並設定按住延遲，讓 Control-C 等常用快捷鍵正常運作。

核心功能

不繞雲端的語音輸入。

01

本機語音辨識

Qwen3-ASR 0.6B 8-bit 在 Apple 晶片上執行，並自動偵測 30 種語言。

02

本機 AI 潤飾

Qwen3.5 0.8B 4-bit 可整理標點、斷句與口語表達，不呼叫雲端寫作 API。

03

即時翻譯

說一種語言，透過 Apple Translation 與已下載的裝置端語言套件輸出另一種語言。

04

自訂按住說話

錄製單鍵或組合鍵並設定按住時間，避免常用快捷鍵誤觸錄音。

05

跨應用程式輸入

放開快捷鍵後，CoveType 將結果插入目前應用程式的游標位置。

06

選單列常駐

小型呼吸燈顯示閒置、聆聽、處理與完成狀態，並可登入後自動啟動。

私隱與安全

清楚易懂的本機處理路徑。

日常轉錄與 AI 潤飾不呼叫雲端語音辨識或寫作 API。只有主動按住已設定快捷鍵後才會開始錄音。

音訊留在本機

暫存錄音在 Mac 上處理，流程完成後清除。

模型留在本機

Qwen3-ASR 與 Qwen3.5 儲存在 CoveType 的本機應用程式資料。

無需帳號與同步

CoveType 不要求登入，也不會同步轉錄歷史。

權限用途有限

咪高峰用於擷取語音；輔助使用權限用於全域快捷鍵及向目前應用程式輸入文字。

何時會使用網路？

首次安裝會下載固定版本的執行環境與模型；macOS 會按需要下載 Apple Translation 語言套件；設定更新通道後，更新檢查可能連線到發佈位址。日常辨識與潤飾仍在本機完成。

匿名每日使用統計

預設開啟，可在用戶端「使用回饋」視窗中關閉。CoveType 每 24 小時最多透過 HTTPS 傳送一次隨機安裝編號、應用程式版本、macOS 版本和處理器架構；Cloudflare 只判斷國家，CoveType 不儲存原始 IP。絕不包含錄音、轉錄結果或輸入文字。

你的本機語音轉文字路徑

  1. 1說話只有按住所選快捷鍵時才開始錄音。
  2. 2本機轉錄Qwen3-ASR 在 Mac 上把語音轉換為文字。
  3. 3本機潤飾可選 Qwen3.5 在裝置端改善可讀性。
  4. 4翻譯或輸入透過 Apple 裝置端翻譯，或將結果插入目前應用程式。

WINDOWS 狀態

目前應用程式不支援 Windows。

目前下載版依賴 macOS 專用 AppKit/SwiftUI、AVFoundation、輔助使用 API、Apple Translation 與 MLX，無法直接複製或重新封裝到 Windows。

目前 Windows 上可使用甚麼？

源碼庫已包含適用於 Windows 10/11 x64 的官方 Qwen3-ASR PyTorch 後端安裝腳本與瀏覽器咪高峰測試頁。它能進行本機辨識測試，但尚不是全系統托盤輸入工具。

完整 Windows 版需要甚麼？

需要獨立開發 .NET 8 桌面客戶端、WASAPI 錄音、Windows 全域快捷鍵與 SendInput、PyTorch/ONNX 模型執行環境、本機翻譯方案、簽署安裝程式，並在真實 Windows 10/11 裝置測試。建議目標：x64、最低 8 GB 記憶體、建議 16 GB、8–10 GB 可用空間；NVIDIA 顯示卡非必要但可明顯降低延遲。

詳細說明

安裝前須知

CoveType 是完整的 macOS 輸入法嗎？

不是。目前版本是選單列語音輸入工具，透過全域按住說話快捷鍵向目前應用程式插入文字，無需切換系統輸入來源。

為何需要輔助使用權限？

macOS 要求此權限才能處理全域快捷鍵並向另一個前景應用程式輸入文字。你可隨時在系統設定撤銷。

本機潤飾支援所有辨識語言嗎？

Qwen3.5 小模型具多語言能力，但 30 種辨識語言的潤飾品質並不完全相同。辨識支援與潤飾品質是兩回事。

所有內容都是開源嗎？

CoveType 應用程式程式碼按 GPLv3 發佈；Qwen 模型、Python 套件與 Apple 框架各自遵循其授權與條款。
